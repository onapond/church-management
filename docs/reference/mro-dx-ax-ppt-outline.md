## 슬라이드 2. 핵심 사항 요약

제목:
- 핵심 사항 요약

본문:
- `Next.js + Supabase + TypeScript strict` 기반 운영 시스템 구축
- 출결, 보고, 승인, 회의, 회계, 알림까지 하나의 흐름으로 통합
- 권한 체계와 RLS를 적용해 데이터 접근을 통제
- AI를 코어에 직접 넣지 않고, 나중에 붙일 수 있도록 AX-ready 구조로 설계
- MRO 환경에서도 구매 요청, 승인, 회의, 후속 작업 자동화로 확장 가능

한 줄 메시지:
- "운영 프로세스를 디지털화하고, AI를 안전하게 붙일 수 있게 준비한 시스템"

## 슬라이드 3. DX 기능 1: 보고/승인 워크플로우

제목:
- DX 기능 1. 보고 작성 및 승인 워크플로우

핵심 포인트:
- 반복 보고 업무를 표준 양식으로 전환
- 임시저장, 제출, 승인 상태를 시스템으로 관리
- 승인 이력과 상태 추적 가능
- 서버 저장 경계 강화로 운영 신뢰성 개선

실무 의미:
- 수기 보고 → 구조화된 디지털 보고
- 구두 승인 추적 → 상태 기반 승인 흐름

스크린샷:
- 보고서 목록 `/reports`
- 보고서 작성 `/reports/new`
- 보고서 상세/승인 `/reports/[id]`

권장 캡션:
- `그림 1. 운영 보고의 작성부터 승인까지 연결된 디지털 워크플로우`

## 슬라이드 4. DX 기능 2: 권한/RLS 기반 운영 통제

제목:
- DX 기능 2. 권한 체계와 데이터 통제

핵심 포인트:
- 역할별 접근 제어 적용
- 부서 범위 기반 데이터 조회 제한
- 클라이언트 UI 제어뿐 아니라 RLS로 DB 레벨 보호
- 권한 로직을 중앙화해 유지보수성 확보

실무 의미:
- 운영 시스템에서 중요한 것은 "기능"보다 "누가 무엇을 볼 수 있는가"
- 실제 서비스 운영 기준의 보안/거버넌스 감각을 반영

근거 코드:
- `src/lib/permissions.ts`
- `src/lib/supabase/server.ts`
- `src/app/api/reports/save/route.ts`

스크린샷:
- 사용자/권한 관리 `/users`
- 승인 대기 화면 `/approvals`

권장 캡션:
- `그림 2. 역할과 부서 범위에 따라 통제되는 운영 데이터 접근 구조`

## 슬라이드 5. AX 기능 1: 구조화된 회의록

제목:
- AX 기능 1. 구조화된 회의록 기반 AI 준비

핵심 포인트:
- 회의 내용을 자유 텍스트가 아니라 구조화된 항목으로 저장
- 논의 내용, 결정 사항, 인수인계 메모 분리
- 기존 코어를 깨지 않고 additive change로 확장
- 향후 요약, 결정사항 추출, Task 추천으로 연결 가능

실무 의미:
- AI는 데이터가 구조화되어 있어야 붙일 수 있음
- 회의 데이터를 AI-ready 형태로 만드는 것이 AX의 출발점

스크린샷:
- 회의 목록 `/meetings`
- 회의 상세 `/meetings/[id]`

권장 캡션:
- `그림 3. AI 요약 및 Task 추천으로 확장 가능한 구조화 회의록`

## 슬라이드 6. AX 기능 2: 알림/자동화 기반 운영 흐름

제목:
- AX 기능 2. 알림과 자동화 기반 운영 흐름

핵심 포인트:
- 승인 단계 변화에 따라 알림 자동 발송
- 인앱 알림 + 웹 푸시 지원
- 수작업 추적을 이벤트 기반 후속 조치로 전환
- 향후 AI 알림 우선순위화, 요약 알림으로 확장 가능

실무 의미:
- 사람 기억에 의존하던 후속 조치를 시스템 이벤트로 전환
- 운영 지연을 줄이고 반응 속도를 높임

스크린샷:
- 상단 알림 패널
- 필요 시 승인 완료/반려 예시 화면

권장 캡션:
- `그림 4. 승인 이벤트와 연결된 자동 알림 흐름`

## 슬라이드 9. 기술 구조 요약

제목:
- 기술 구조 요약

본문:
- Frontend: `Next.js 16`, `React 19`, `TypeScript strict`
- Data/Auth: `Supabase`, `RLS`, `Storage`, `Realtime`
- State: `TanStack Query`
- Notification: `web-push`
- Deployment: `Vercel`

구조 요약:
- 얇은 페이지 구조
- 중앙 권한 함수
- 서버 라우트 검증
- DB 저장 경계 강화
- AI는 plugin/add-on 방향

권장 시각자료:
- 간단한 아키텍처 박스 다이어그램 1개

## 슬라이드 10. 결론

제목:
- 결론

본문:
- 이 프로젝트는 실제 운영 업무를 디지털화한 시스템이다.
- 단순 화면 개발이 아니라 권한, 승인, 저장 신뢰성, 알림까지 고려했다.
- AX는 "AI 기능 추가"가 아니라 "AI를 붙일 수 있는 구조 준비"로 접근했다.
- MRO 환경에서도 구매, 승인, 회의, 공급사 협업, 후속 Task 관리로 확장 가능하다.

마무리 문구:
- "운영 시스템 구축 경험과 AX-ready 설계 경험을 동시에 보여줄 수 있는 사례"
