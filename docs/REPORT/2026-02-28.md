## ✅ 완료한 작업

### 1. 셀장보고서 결재 라인 단순화

- **요구사항**: 셀장보고서 결재를 회장/부장/목사 없이 cu1 팀장이 직접 최종결재
- **구현**:
  - `ReportDetail.tsx` — `checkApprovalPermission()` 함수에 `cell_leader` 분기 추가
    - `submitted` 상태 + cu1 `is_team_leader=true` → `'final'` 반환
  - `permissions.ts` — `canApprove()`에 `'team_leader'` 추가
  - `approvals.ts` — 팀장 결재함: 부서 셀장보고서 `submitted` 상태만 표시
  - `ApprovalsClient.tsx` — 팀장 역할 표시 + "셀장(제출) → 팀장(최종결재)" 안내
- **관련 파일**: ReportDetail.tsx, permissions.ts, approvals.ts, ApprovalsClient.tsx

### 2. 셀장보고서 팀장 알림 버그 수정

- **버그**: 셀장이 보고서 제출해도 팀장에게 알림이 오지 않음
- **근본 원인**: `ReportForm.tsx`에서 `createApprovalNotification()` 호출 시 `reportDepartmentId` 누락
- **수정**: `reportDepartmentId: form.department_id` 추가
- **알림 흐름**:
  - 셀장보고서 제출 → `user_departments` 테이블에서 해당 부서 팀장 조회 → 팀장에게 알림
  - 셀장보고서 최종결재/반려 → 작성자에게 알림
- **관련 파일**: `src/components/reports/ReportForm.tsx`, `src/lib/notifications.ts`

### 3. 심방보고서(visitation) 타입 추가

- **요구사항**: 보고서 작성 메뉴에 심방보고서 유형 추가
- **구현**:
  - DB: `report_type` PostgreSQL enum에 `'visitation'` 값 추가 (MCP 마이그레이션)
  - `ReportForm.tsx`: 심방 대상자, 날짜, 심방 내용(`main_content`), 기도제목(`application_notes`) 폼

  - `ReportListClient.tsx`: 심방보고서 탭(🙏) 추가, `bg-rose-100` 아이콘 배경
  - `types/database.ts`, `reports/new/page.tsx`, `EditReportClient.tsx`: visitation 타입 지원
- **결재**: 일반 보고서와 동일한 결재 라인 (회장→부장→목사)
